## Fuzz Target Inventory

This work item adds **performance benchmark infrastructure** (criterion-based). Fuzz testing for the core diffguard libraries is already implemented in the `fuzz/` crate. The benchmark infrastructure itself uses synthetic in-memory inputs generated by fixture functions, so traditional fuzzing is not applicable to the benchmark code.

### Fuzz Targets (existing in `fuzz/` crate)

| Target | File | Purpose |
|--------|------|---------|
| `unified_diff_parser` | `fuzz/fuzz_targets/unified_diff_parser.rs` | Exercises `parse_unified_diff()` with structured diff-like inputs |
| `evaluate_lines` | `fuzz/fuzz_targets/evaluate_lines.rs` | Focused fuzz target for `evaluate_lines()` with arbitrary rule/line inputs |
| `rule_matcher` | `fuzz/fuzz_targets/rule_matcher.rs` | Complex fuzz target combining evaluate_lines, preprocessing, and suppression |
| `preprocess` | `fuzz/fuzz_targets/preprocess.rs` | Exercises `Preprocessor::sanitize_line()` with language-aware fuzzing |
| `config_parser` | `fuzz/fuzz_targets/config_parser.rs` | Exercises TOML config parsing pipeline |
| `regex_pattern` | `fuzz/fuzz_targets/regex_pattern.rs` | Fuzzs regex pattern compilation |
| `glob_pattern` | `fuzz/fuzz_targets/glob_pattern.rs` | Fuzzs glob pattern compilation |

### Fuzz Target Details

#### 1. unified_diff_parser.rs
- **Inputs:** Structured `DiffInput` with scope selector, files, hunks, and lines
- **Generates:** Diff strings with various file types (binary, deleted, renamed, normal)
- **Assertions:** 
  - `stats.lines == lines.len()`
  - `line.line >= 1` for all lines
  - `!line.path.is_empty()`
- **Scope variants tested:** Added, Changed, Modified, Deleted

#### 2. evaluate_lines.rs
- **Inputs:** Up to 200 `FuzzLine` inputs with path, line number, content
- **Rules:** Up to ~50 `FuzzRule` inputs with various patterns
- **Assertions:**
  - `eval.findings.len() <= max_findings`
  - `total_counts >= eval.findings.len()`
  - `truncated_findings + findings.len() == total_counts`
  - `eval.lines_scanned == input_lines.len()`

#### 3. rule_matcher.rs
- **Inputs:** Up to 100 `FuzzLine` with optional suppression directives, up to ~50 `FuzzRule`
- **Tests:** evaluate_lines + preprocessing + suppression parsing + SuppressionTracker
- **Assertions:** Same as evaluate_lines plus length preservation for preprocessed output

#### 4. preprocess.rs
- **Inputs:** Language enum, PreprocessOptions (mask_comments, mask_strings), up to 32 lines
- **Tests:** All 12 language variants with both options
- **Assertions:**
  - `result.len() == input.len()` for all lines
  - Length preserved after language switch
  - Reset behavior consistent

#### 5. config_parser.rs
- **Inputs:** Structured TOML generation OR raw byte fuzzing
- **Tests:** Raw TOML parsing, ConfigFile deserialization, RuleConfig compilation, enum parsing
- **Assertions:** None (error handling only — should not panic on invalid input)

#### 6. regex_pattern.rs
- **Inputs:** Raw bytes interpreted as regex pattern string
- **Tests:** Pattern compilation via `compile_rules()`
- **Assertions:** None (error handling only)

#### 7. glob_pattern.rs
- **Inputs:** Raw bytes interpreted as glob pattern string
- **Tests:** Glob compilation via `compile_rules()` for both include and exclude paths
- **Assertions:** None (error handling only)

## Fuzzing Strategy

### Scope of Fuzzing for This Work Item

The benchmark infrastructure (`bench/` crate) is **not fuzzed directly** because:

1. **Benchmark code is deterministic** — Uses fixed synthetic inputs generated by `fixtures.rs`
2. **No external input parsing** — All inputs are generated programmatically in-memory
3. **Criterion framework handles measurement** — The benchmark framework itself is assumed correct

However, the **code being benchmarked** (parsing, evaluation, rendering, preprocessing) IS fuzzed via the existing fuzz targets in `fuzz/`.

### Key Fuzzing Invariants Verified

| Property | Verified By |
|----------|------------|
| `parse_unified_diff` never panics | `unified_diff_parser.rs` |
| `evaluate_lines` findings count bounded | `evaluate_lines.rs`, `rule_matcher.rs` |
| `evaluate_lines` counts consistency | `evaluate_lines.rs`, `rule_matcher.rs` |
| Preprocessor output length preservation | `preprocess.rs`, `rule_matcher.rs` |
| Config parsing never panics | `config_parser.rs` |
| Regex/glob compilation never panics | `regex_pattern.rs`, `glob_pattern.rs` |
