# Specs: Fix Incorrect Assertions in edge_case_snapshot_tests

## Feature/Behavior Description

Fix 4 incorrect test assertions in `edge_case_snapshot_tests.rs` that fail even when the renderer implementations are correct. These tests validate that various output formats (JUnit XML, CSV, TSV, SARIF) correctly render findings with empty fields and Unicode characters.

## Acceptance Criteria

1. **`test_junit_empty_fields`:** Change lines 361-362 from `xml.contains("<checkstyle")` and `xml.contains("</checkstyle>")` to `xml.contains("<testsuites>")` and `xml.contains("</testsuites>")` respectively.

2. **`test_csv_empty_fields`:** Change line 433 from `lines[1].contains(",,,,")` to verify the actual output pattern `",0,,warn,,"` where `0` is the line number and `warn` is the severity.

3. **`test_tsv_empty_fields`:** Change line 501 from `lines[1].contains("\t\t\t\t\t")` to verify the actual output pattern `"\t0\t\twarn\t\t"` where `0` is the line number and `warn` is the severity.

4. **`test_sarif_unicode_characters`:** Change line 230 to add `&` character to the message: `message: "错误消息 with emoji 🎉 &".to_string()` so that the assertion `json.contains("&#x") || json.contains("&amp;")` on line 244 can pass.

5. After all fixes, running `cargo test -p diffguard-core -- edge_case_snapshot_tests --ignored` passes all 4 tests.

## Non-Goals

- No changes to implementation code (renderers are correct)
- No changes to test structure or helper functions
- No changes to other tests in the file

## Dependencies

- Issue #79 (tests are currently `#[ignore]`d pending this issue)
- Renderer implementations must remain correct:
  - `junit.rs` uses `<testsuites>` as root
  - `csv.rs` uses 5-commas format `"{},{},{},{},{},{}\n"`
  - `tsv.rs` uses 5-tabs format `"{}\t{}\t{}\t{}\t{}\t{}\n"`
  - `sarif.rs` escapes `&` as `&amp;`
